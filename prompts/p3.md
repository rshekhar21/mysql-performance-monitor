Before claiming this application is complete, perform a full production-readiness and end-to-end audit.

Do not rely only on TypeScript compilation, linting, unit tests, or successful builds.

1. Inventory every page, route, button, form, table action, API endpoint, and collector job.
2. Identify:
   - placeholder pages;
   - buttons without handlers;
   - routes without components;
   - frontend actions without backend implementation;
   - backend endpoints with no frontend usage;
   - mock or hard-coded data;
   - incomplete loading, error, and empty states.
3. Test the complete user workflow:
   - unauthenticated visitor is redirected to login;
   - valid login succeeds;
   - invalid login fails safely;
   - logout clears authentication;
   - protected routes cannot be opened without login;
   - add monitored server form opens;
   - form validation works;
   - connection test works;
   - server can be saved;
   - server appears in the list;
   - credentials are encrypted and never returned;
   - collector detects the server;
   - collector stores snapshots;
   - overview displays real collected data;
   - charts load real historical data;
   - databases and table sizes load;
   - running queries load;
   - errors are shown safely when MySQL is unavailable;
   - server edit, enable/disable, test, and delete work where supported.
4. Test fresh-browser and incognito behavior.
5. Test direct navigation to React routes such as /servers.
6. Test production Docker deployment, not only local npm development.
7. Test through a reverse proxy where frontend and API share one HTTPS domain.
8. Ensure frontend production API URL is configurable and never defaults to localhost in production.
9. Ensure Docker Compose does not expose the app database on host port 3306.
10. Ensure API, collector, and web host ports are configurable to avoid VPS conflicts.
11. Include a production Nginx SPA fallback.
12. Add Docker health checks for app-db, API, collector, and web.
13. Verify startup order and graceful recovery after container restart.
14. Create automated integration or end-to-end tests for critical workflows.
15. Run:
    - lint;
    - formatting check;
    - typecheck;
    - unit tests;
    - integration tests;
    - end-to-end tests;
    - production builds;
    - Docker Compose build and startup;
    - migrations;
    - seed;
    - health checks.
16. Produce a completion matrix showing every feature as:
    - implemented and tested;
    - implemented but untested;
    - incomplete;
    - intentionally deferred.
17. Do not state that the app is complete while any essential workflow is incomplete or untested.
18. Update the deployment documentation with a tested CloudPanel + Docker Compose procedure.
19. Report every failure honestly, including the exact command and error.
20. Do not commit until the audit and required fixes are complete.
